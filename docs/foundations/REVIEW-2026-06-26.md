# Benchmark-suite review — synthesis (2026-06-26)

Four independent read-only review agents (package/CLI UX · distribution/deploy · methodology
robustness · run-flow+timeframes). Scoped to the owner decision: **the suite = Agentic + Knowledge
+ Instruction + Tool-calling in one run; Math + Long-Context removed; Coding TBD.**

## Bottom line
The **scoring/stats engine is genuinely strong** and unusually self-aware: seeded bootstraps,
sha256-frozen item sets, a real `scorecard_id` over the axis registry + scorer versions, a registry
drift-gate, a reasoning-leak conformance gate, chance-correction with provenance. **Knowledge
(MMLU-Pro) + Instruction (IFBench) are solid and reproducible.**

**The risk is concentrated in one place: the agentic axis — 70% of the headline Index, and what you
just put at the center.** It is the least deterministic and least-frozen part of the system, shipped
ahead of its own repeatability validation. The live #1/#2 result (Qwen 32.78 vs Gemma 32.47, a
**0.31-pt gap**) may be **within agentic run-to-run noise**. This is the "if this piece isn't robust,
the whole thing comes crashing down" risk — and it's real, but fixable.

Everything else (deploy/packaging) is **operational, not architectural**.

---

## A. Robustness & repeatability — the load-bearing question

**Strong:** judge-free deterministic scoring of fixed transcripts; seeded CIs; frozen item hashes;
scorecard identity so a reweight bumps identity instead of silently re-scoring; conformance gate
demotes runs that leak reasoning.

**The problems — all on the agentic axis (and the docs):**
1. **Not repeatable.** Two production nondeterminism sources: per-block wall-clock timeouts
   (`sandbox.py:49`, `config.py:19`) make pass/fail **machine-load-dependent**; unbounded
   conversation history (`protocol_c_loop.py:190`) causes late-turn context-shift drift. At raw ASR
   ×0.70, the axis's *noise* maps ~1:1 onto rank — a few of 96 tasks flipping on a timeout can
   reorder #1/#2.
2. **Not frozen.** AppWorld-C is **excluded from the suite-freeze** (`test_axes_registry.py:69`);
   tasks are picked at runtime from whatever AppWorld install is present, hash computed on the fly,
   AppWorld version/data/`evaluate()` not in the scorecard. Contradicts the "frozen reproducible
   suite" claim for the 70% axis.
3. **`<think>`-leak scorer bug CLASS.** No scorer strips reasoning tags; the tc_json 0% bug has
   **siblings**: `mcq.py` and `math_symbolic.py` can be corrupted by letters/`\boxed{}`/"answer:"
   inside a `<think>` block. The conformance gate only demotes the whole run above 2/10/25%
   thresholds; sub-threshold leaks silently corrupt items while the run still ranks. **No
   scorer-level `<think>` test exists** — the gap that let tc_json ship.
4. **Infra failures counted as model failures** on agentic — a timed-out block is indistinguishable
   from low capability; at 70% weight, a few infra hiccups swing the headline.
5. **Docs misrepresent the shipped product.** `METHODOLOGY-v1.2-LOCKED.md` (the canonical doc) says
   headline = K+I 0.5/0.5; **code ships agentic 0.70/0.15/0.15**. The v2.0 scaling transform
   (`A*=ASR/50…`) is **unimplemented and unhashed** — a silent-rescore hole the moment any
   normalization is added.

## B. Deployable one-command package — operational, no redesign needed
- **Install is fiction today:** `pipx install localbench` resolves nowhere; README even claims "no
  git clone required" (false). No wheel is built/hosted (#30). *Good news:* the suite is now
  **bundled in the wheel** (old data-access blocker fixed).
- **Bring-your-own-server:** localbench never serves the model; the user must run
  vLLM/llama.cpp/Ollama and pass `--endpoint`. Under-scaffolded (no `localbench serve`, no examples).
- **No "get on the board" path** — `submit pack` is offline-only, needs an unexplained signing key,
  no upload/ingest endpoint.
- **Reproducibility leaks:** core deps all **floating** (no lockfile) → scores drift as deps update.
- **Code license undecided + `cli/` has no LICENSE** → hard blocker for publishing a wheel.
- **Default `--lane answer-only` ≠ headline `capped-thinking`** → a copy-paste user silently gets
  non-board-comparable scores.

## C. The new 4-axis scope — implications
- **Agentic in the one command** ⇒ the AppWorld **sandbox becomes a hard prerequisite** (today
  WSL+bubblewrap; the CLI can't even import it) and runs **hours**. Mitigation: **Dockerize the
  sandbox** (install Docker → one command). Runtime is inherent (multi-hour, BYO-GPU).
- **Tool-calling now counts** ⇒ Index becomes a **4-way weighted split**; the 0%-weight gate /
  `ConformancePill` framing is retired — it gets a normal weighted axis cell.
- **Doing this right REQUIRES fixing agentic robustness (A) first** — otherwise the one command
  produces an unreproducible headline.

## Proposed priority order
1. **Reconcile docs↔code; decide raw-vs-scaled composite** (cheap; removes the worst credibility hole).
2. **Agentic robustness:** freeze AppWorld-C items + fold its harness/version into the scorecard;
   fix timeouts (CPU-relative/retry) + history windowing/strip-thinking; **measure repeatability**;
   set the agentic weight by the result.
3. **Scorer `strip_reasoning()` + per-scorer `<think>` regression matrix** (kills the bug class).
4. **Re-weight the Index** for the 4-axis suite; remove Math + Long-Context from the registry.
5. **Package for deploy:** pick+ship the code license; pin/lock deps; build+host the wheel
   (defer R2 — suite's bundled); `serve` helper + `doctor --endpoint`; default lane → capped-thinking;
   honest README; a real submission path.
6. **tool-eval-bench integration** — now a *weighted axis*, not a gate.
7. **Landing flowchart + timeframe.**
8. **Dockerize the agentic sandbox** for turnkey external deploy.
