## 0. Where the project is right now

local-bench is a **reproducible, judge-free local-LLM leaderboard**. As of this session the headline metric is the
**agentic-led Local Intelligence Index v2.0 = 0.70·Agentic + 0.15·Knowledge + 0.15·Instruction**
(Agentic = AppWorld-C interactive API-coding ASR; Knowledge = MMLU-Pro chance-corrected; Instruction = IFBench; all 0–100).

- **2 ranked models:** Qwen3.6-27B **32.78 (#1)**, Gemma-4-31B **32.47** — a **statistical tie** (CIs overlap heavily; N=96 on agentic).
- Models run locally on one **RTX 5090 (32 GB)**, served via llama.cpp (Core Text) and vLLM (the Qwable NVFP4 run).
- Site = Next.js **static SSG** (`web/`), a **pure renderer** of `cli/runs/board/board_v2.json` via `web/build_data.py` → `web/public/data/`.
  The site does **not** auto-update; you regenerate data (`web/build_data.py`) and rebuild (`scripts/build-site.ps1`).

## 8. PIPELINE (how a new benchmark reaches the site)

`localbench run` (Core Text) **or** the AppWorld-C funnel (Agentic) → run JSON → curate in `web/data_sources.json` →
`localbench board` → **`board_v2.json`** (immutable, the single source of truth) → `web/build_data.py` → `web/public/data/` →
`scripts/build-site.ps1` (`next build`) → static site → (deploy, gated). **Curated + static, not live.**
